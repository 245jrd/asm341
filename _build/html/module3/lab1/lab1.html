Lab 3.1: Bot Protection¶

In this lab we will investigate some of the features that comprise F5’s Advanced WAF.

Task 1 - Configuring Bot Defense¶

  1. Browse to the BIGIP managemnet console.

  2. Create a New ASM policy by navigating to Security -> Application Security -> Security Policies

  3. Click Create New Policy and fill in the page as follows, then click Create Policy

  4. Now lets create a DoS profile to go along with our WAF policy

  5. Navigate to Security -> DoS Protection -> Dos Profiles and then click the Create button.

  6. Click ‘Create’ to create a new DoS profile

  7. Name the profile “module3” and click ‘Finished’

  8. Click the profile to configure.

  9. Select the ‘Application Security’ tab

  10. Click ‘Edit”, followed by the ‘Enabled’ checkbox to turn on Bot Detection.

  11. Let’s configure the types of bot protection that offer the best bang for the buck, starting with Signatures. Click the ‘Bot Signatures’ tab and click edit.

  12. Now click the ‘Enabled’ checkbox. You’ll see we can group bots by category or can select them uniquely.

  13. Click ‘Edit’ to explore the settings further. Try to resist the urge to modify any settings, the defaults will serve us well.

  14. For complete protection, let’s go ahead an enable Proactive Bot Defense as well. Click on the ‘Proactive Bot Defense’ tab, click ‘Edit’, and change the dropdown to “During Attacks”.

  15. Click ‘Update’ to save changes.

  16. Now let’s bind this DoS policy to a Virtual Server. Navigate to Local Traffic -> Virtual Servers -> Virtual Server List and select ‘asm_vs’.

  17. Click on ‘Security” tab and select ‘Policies’

  18. Enable the ‘Module3’ Application Security and DoS Protection profiles.

  19. Click ‘Update’

Task 2 - Simulating Bot Traffic¶

  1. Now that we have a DoS profile in place, lets test it!
  2. Open a command prompt on your jumpbox.
  3. Execute the following command:
    python /opt/goldeneye.py http://10.1.10.145 -d -w 50 -s 200
  4. Wait 5 minutes for traffic generation and logging.
  5. In the BIG-IP WebUI, Navigate to Security -> Event Logs -> Application Events
  6. Review the attacks detected by ASM.
